# 01 · 产品愿景与玩法

> 工作代号 **《小镇有灵 / Living Town》**（可改）。idea 起点：**类像素版「模拟人生」，集成 LLM**。
> 本文把这个 loose idea 收敛成一个**能落地、能复用现有流水线、能凸显 LLM 价值**的具体产品。

## 1. 一句话愿景

> 一座**会自己活起来**的像素小镇：每个居民由 LLM/SLM 赋予**记忆、性格、一天的计划与即兴的对话**，
> 玩家作为新搬来的居民，**和有灵魂的 NPC 交往、并见证涌现的社会故事**。

它不是「带聊天框的模拟人生」，而是：**模拟人生的需求/作息系统当骨架，generative-agents 的记忆/反思/社交当灵魂，LLM 当声音**。

## 2. 为什么是这个 idea（与 LLM 的契合）

LLM 在游戏里**最不擅长**的是数值最优化与硬规则（《小鱼岛》doc 13 已论证），**最擅长**的是：
人格化的语言、记忆驱动的一致性、对开放情境的即兴反应、涌现的社会关系。
而「模拟人生」恰恰是一个**语言与社会**的游戏，不是一个**数值最优**的游戏——所以 LLM 的长处正好压在玩法的爽点上，短处可被确定性引擎兜住。
这让本项目成为**展示「本地 LLM + 游戏」**的理想载体，且能复用 22nd 已跑通的全部 LLM 管线。

## 3. 核心循环

### 3.1 世界循环（每个游戏日）
```
清晨：每个 Agent 用 LLM 生成/复用一天的「粗规划」（起床→工作→社交→吃饭→睡觉）
全天：world tick（需求衰减、移动、寻路、对象交互）以确定性引擎高频推进；
      Agent 多数时刻执行「当前 option」的廉价脚本动作（不调模型）；
      当发生「有意思的事件」（遇到人、需求告急、导演点名、玩家介入）才异步调一次 LLM；
      两个 Agent 相遇 → LLM 生成简短对话 → 双方写入记忆流。
夜晚：每个 Agent 触发一次「反思」（把当天记忆 LLM 提炼成更高层洞察，写回记忆），睡觉。
```

### 3.2 玩家循环
- **自由对话**：走到任一 NPC 旁，输入文本 → NPC 基于其人设 + 记忆 + 当下世界状态用 LLM 回应（流式 + 思考动画藏延迟）。
- **观察与八卦**：旁观 NPC 间的对话/关系变化；翻看某 NPC 的「记忆/心事」面板（其记忆流摘要）。
- **介入**：送礼、撮合/拆散、传播谣言、办活动（如 Stanford 著名的「情人节派对」涌现——发起一个意图，让它经由 NPC 间对话与记忆自行扩散）。
- **目标（可选轻量）**：融入小镇（提升与各 NPC 的好感）、解锁 NPC 的背景故事、达成若干「社会成就」（撮合一对、化解一场争执、举办一次全员到场的活动）。

## 4. 玩家体验支柱

1. **活人感（believability）**：NPC 记得你昨天说的话、记得彼此的恩怨，行为与性格一致——这是 Stanford 记忆流+反思的产物，是本作的灵魂。
2. **涌现（emergence）**：没有写死的剧情，社会戏剧从 Agent 互动里长出来；每周目不同。
3. **零门槛即玩、渐进增强**：**无模型也能玩**（确定性脚本生活模拟全程可玩），有本地/远程模型则 NPC「活」起来——LLM 是 progressive enhancement，不是硬依赖。这直接继承 22nd 的「引擎兜底」纪律。
4. **治愈/低压**：像素治愈风、慢节奏、可加速/暂停；不是生存压力游戏。
5. **可解释/可复现（开发期）**：沿用确定性种子 + scenario 回放，让「涌现」也能被测试与评审（见 doc 02 的 bench 红线）。

## 5. 范围与「不做什么」（务实收敛）

**首版做（M1–M4，见路线图）**：
- 一张小地图（~1 屏可滚动）、**6–10 个 NPC**、每 NPC 一份人设 + 需求 + 作息。
- 确定性需求/效用生活模拟（The Sims 式 motive→advertised interaction）跑通且好看。
- LLM 对话（玩家↔NPC、NPC↔NPC 简短对话）+ 记忆流 + 夜间反思 + 一个轻量「导演」。
- 三档 LLM 后端复用 22nd（logic / LM Studio / 内置 SLM）。

**首版不做（明确砍掉，控范围）**：
- ❌ 大世界/多镇、几百个 Agent（Dwarf-Fortress 级别——留作 Bevy 逃生舱，见 doc 02）。
- ❌ 语音 STT/TTS（Mantella 证明可行，但首版不做，留作打磨期可选）。
- ❌ 建造/装修/经济大系统（先把「活人 + 社交」做透，再加沙盒）。
- ❌ 联网/多人。
- ❌ 复杂战斗。

> 收敛原则（取自 22nd 的成功经验）：**先把最有魅力、也最依赖 LLM 的「活人社交」做透，其余沙盒系统作为后续内容深度分档（S/M/L）逐步加。**

## 6. 与《小鱼岛》的异同（一表看清继承与新增）

| 维度 | 小鱼岛（22nd） | 小镇有灵（26th，新增挑战） |
|---|---|---|
| 类型 | 回合制经营/构筑卡牌，对 1 个 AI | 准实时多 Agent 生活模拟，N 个 LLM NPC |
| LLM 角色 | AI 对手：在合法候选里挑 + 吐槽 | NPC 的大脑/嘴：对话 + 规划 + 反思 + 社交（更重） |
| 调用频率 | 我方回合一次（~秒级可接受） | 必须**稀疏异步**：导演-Agent 分层、option-action、缓存，否则成本/延迟爆炸 |
| 记忆 | 几乎无（局部状态） | **记忆流 + 反思树**是核心（Stanford） |
| 确定性 | 强（卡牌、种子回放） | 仍要（开发期 bench），但叠加涌现——靠 scenario+seed 守住 |
| 复用 | —— | 引擎/导出/serve/judge/SLM 后端/美术回退 **全部复用** |
