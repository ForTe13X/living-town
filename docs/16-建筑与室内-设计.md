# docs/16 · 建筑（高层）与室内（多样复杂）— 设计与 roadmap

> 2026-07-03。方法：4 线并行设计（空间模型 / 室内内容 / 渲染 / 涌现与规模）+ 每线独立对抗校验（读码核对行号、确定性红线、CC0 素材实证）。校验修正已吸收进本文。
> **一句话结论**：室内**不是**新空间层/新物理，而是"把 `_area_at` 的 5 个粗区细化成带 `enclosed`/`owner`/`type` 语义的房间集 + 一个纯渲染的屋顶切除层"。四线独立收敛到同一句：**别给 `pos` 加 z/floor（那是回放杀手），把楼当成同一张平面网格上多划的房间矩形。**

---

## §0 全案基石（四线一致，读码确认）
**`digest`（bench/Invariants.gd:373）只序列化 `event_log` 的 `id/type/actor/target/accepted/subject/tick`——`pos`/`floor`/`map` 根本不进 digest。** 所以"agent 在哪层哪个室内格"本身是**纯呈现态**：只要不改变事件流的顺序与内容，加室内维度对字节级复现**零影响**。这把三条候选空间模型全部从"回放杀手"降级为"呈现 + 邻近语义"问题。真正受影响的只有三处：
1. `_nearby_agents`（谁能撞见谁 → 社交/目击事件流）——这是唯一进 digest 的语义变化；
2. LOD 焦点距离（room 不参与，`_is_far` 只看 pos↔focus，天然"整栋楼离焦点远→楼内全 far"）；
3. 跨层寻路（若做真多层）——不得引入 live RNG/浮点/字典插入序。

## §1 空间模型：房间矩形，不是 z 坐标
| 方案 | 做法 | 判定 |
|---|---|---|
| **A 复合位置 `location={map,floor,pos}`** | 室内=独立子网格，agent 带 map 维，楼梯=portal 节点图 | **多层终态**。语义最干净（真 Z/真隔离），但改动面最大（凡读 `ag["pos"]` 处约 1366/1373/1801/695… 全要过 `loc.pos`）；不宜作第一刀 |
| **B 房间=同平面多划的 area/room 矩形** | 建筑=大 rect，内部切子 rect（`enclosed` 房间）；`_nearby` 的"同区=邻近"直接白嫖成"同房间=邻近"的隐私语义 | **首切片首选**。`_area_at/_move_agent/LOD/寻路/goto_tick` 一行不改，复用一切 |
| **C portal 传送到离屏坐标块** | door→偏移到 x≥1000 的室内区 | **不推荐**。离屏坐标污染 LOD 曼哈顿距离→室内 agent 被误判 far 集体降级；两头不讨好 |

**分两阶段**：先 B（细化房间 + 隐私语义），验证"可进入 + 私密"玩法后，再上 A 做真多层（楼层=独立子网格，楼梯 portal 候选须 sorted 定序 + `_aid` tie-break）。**绝不走 C。**

## §2 为什么现在做：室内是三个已跑机制正缺的那块地板
最强的理由不是"造室内需求"，而是**已存在的机制正缺一块地板**：
- **S3 秘密博弈（confide/leak，已落地在跑）**：当前 confide 发生在"同区"，而最大的区是**开放广场**——"在广场当众吐露心事然后指望不外泄"是叙事漏洞。**enclosed 房间是 confide 唯一物理上说得通的发生地。** 这是最高价值、最小改动的切入点。
- 家访 / 私宴：`visit` = `invite` 的孪生动作，复用 commitment 全套；私宴 = festival 的 `spawn_object` 摆进 home 区而非 plaza（天气门"雨天躲室内"复用）。
- 职场 co-presence（**未落地，规划项**）：workplace = enclosed 房间，同事日日同房间 → 关系/派系加速。
- 住房产权（docs/15 标注为 **Wave 3c 低优先 P 级、等贫富分化成形**，**非已落地**）：room = 产权自然单位，`owner` 字段 + Ledger 记一笔租金；止步于此，**不建租房市场撮合**。

> 校验纠正：不要夸大成"三条已落地的线同时在等"——真正落地的只有 S3 confide/leak 一条；workplace/housing 都是未落地规划项，作 A/B 阶段的可选前哨即可。

## §3 室内内容：编译期展开，不手搓每一间
- **家具 = 既有 object + advertise**（"读书/做饭/在家歇着"= 新 advertise 条目，need 全用既有 fun/energy/…，**零新 need、零新 executor**）。
- **注入方式 = 静态编译，绝不走 `spawn_object`**（后者会 `_log_event('world',...)` → digest 漂）：`_compile_buildings()` 像 `jobs` 的 `extra_advertises`（Sim.gd:226）那样**直接写 `world["objects"][oid]=od`、零 event**。
- **多样性 = 程序化房型模板 + `_hash01(building+persona+职业+财富档)` 选变体**（纯确定，禁 `_rng_at`/`randf`——那是 per-tick 决策流），比手搓省 ~10× 数据量。手写房作"逃生舱"（variant:"literal"）与模板共存。
- **成本红线（thread content 给数）**：`_object_candidates`（Sim.gd:779）每 agent 每 tick 遍历**全部** object×advertise。8 object≈10 advertise；若室内撑到 200 家具≈250 advertise，100 NPC=25000 内层迭代/tick，**吃掉 LOD 省下的算力**。→ 必须捆绑 **D 空间裁剪**：建 `_objects_by_area` 索引，`_object_candidates` 只遍历 agent 所在区（及相邻区）的家具。

## §4 渲染：屋顶切除，与 sim-LOD 同源
| 方案 | 判定 |
|---|---|
| **A 屋顶切除 roof-cutaway（星露谷式）** | **首选**。近端有人在建筑 footprint 内→隐去屋顶露室内、画地板+墙+家具；否则画屋顶盖住。**直接读 `Sim._near_set`/`_is_far`**——同一距离函数同时驱动 sim-LOD 与 vis-LOD（docs/12 §5.2），far 建筑屋顶闭合+零室内 draw，省 CPU 又省 draw item |
| B 楼层切换（模拟人生平面图） | 表现力强但割裂"按距离连续显形"；留作后期室内很深时升级 |
| C 剖面/PIP 小窗 | 是"看板"不是"可进入世界"，可作补充 |
| D 伪等距 2.5D | 推倒重来、与现有正交顶视角+CC0 素材全冲突，SKIP |

- **多层 = 楼层选择器 UI 变量 `active_floor`（纯 WorldView，goto_tick/start_new 不碰）**，不是真 3D 堆叠。
- **首切片零新增 PNG**：复用 `_draw_bed/_draw_stove` 程序化像素家具范式 + `hut` 贴图当屋顶。
- **素材许可（thread render 查证）**：真瓦片升级只用 **CC0**（Ritpop "Modern Houses" TopDown / Shade PunyWorld，引入前必须走 `tools/fetch_assets.py` 落 LICENSE.txt 实证）；**绝不入库** LimeZu Modern Interiors（非 CC0）、Bonsaiheldin Interior（CC-BY-SA 传染式）、2DPIXX（CC-BY）——违"无版权风险"铁律。

## §5 规模：室内是红利不是负担
- **`_nearby` 从"同区 O(区人²)"降到"同房间 O(房人²)"**——房间比区小，社交对枚举天然缩减。
- "整栋楼离焦点远→楼内全 far 聚合"是 `_is_far` 已有行为的自然结果，零改动接住 LOD。
- **relationships 稠密化天然稀疏**：edge 只在同房间生成 → 小圈子 → 缓解 docs/15 §2.4 的"N=5000 稠密 15GB"内存炸弹。
- 高密度：一栋楼=一个 enclosed 房间集，楼内几十人小 footprint 高 N，用 A 之上"共享 far 聚合"模拟而不改坐标维数。

## §6 确定性守法点（落地必守）
1. `_room_at` 与 `_area_at` 同范式纯矩形遍历，**rooms 是独立 dict**（不嵌在 areas 内→无 overlap→无 `_area_at` 首命中的插入序陷阱）；遍历序=JSON 书写序（加载即定序）。
2. `ag["room"]` 缓存必须在**所有** `ag["area"] =` 写点同址一并刷新（grep 全部写点，~360/spawn/init/_move_agent），否则 area 已刷 room 未刷的中间态。
3. **家具编译只在 `_load_data` 一次** + `_buildings_compiled` guard（仿 `_jobs_injected` Sim.gd:224）；`start_new` 不碰 `bld_` 对象（既不清也不重注，防 goto_tick 反复 start_new 翻倍）。
4. 家具展平进 `world["objects"]` 必须 **sort(oid) 后插入**——`_object_candidates` 按插入序遍历 + `_best` 平局取首个 → 插入序影响决策 → 影响 event。
5. **off 门 = 顶层数据门** `if not world.get("rooms",{}).is_empty():` 包住整个 enclosure 新逻辑；rooms 空→完全跳过走旧"任意同区"路径→**逐字节不变**（`Compare-Object` 实测比对，非口头声称）。注意本方案 enclosure 硬编进内建 `_social_candidates`（不走 ext 挂点），off 保证来自"数据缺失"而非"ext=null"。
6. enclosure 门**别写成硬前置**（rooms 空时 confide 归零=破 off 门）；且用 `confide_events` 计数做**阶段1 必过量化门**——卧室人流稀，硬门易饿死秘密博弈，优先"提升 confide 倾向(加分)"或谨慎硬门 + 校准。
7. LOD-off 与 LOD-on **都要验隐私**：far agent 不跑社交，"隔房不目击"可能是"根本没在场"的假阳性→隐私上真机开 LOD 后静默消失。
8. 屋顶切除/楼层选择/滞回全是**纯渲染态**，只读 `_near_set`（确定量），绝不用 `Time`/`randf`，绝不写回 Sim（否则玩家看哪就改历史→破 goto_tick 无玩家介入前提）。

## §7 落地 roadmap（每步五件套门）
**阶段 0 前哨（零空间改动）**：`visit` = `invite` 孪生（登门戏剧），私宴 = festival objects area 设 home。门：off 逐字节 + `type=visit` commitment 正常解算。

**阶段 1 最小室内（核心，先做）**：map.json home 区加 1 个 `enclosed` 卧室 room（rect 须实际覆盖 bed_1[3,3]/bed_2[6,4]，示例 [2,2,3,3] 漏了 bed_2）；`_room_at` + `ag["room"]` 全写点刷新；confide/leak 候选加 enclosure 门（顶层 `if not rooms.is_empty()` 包裹）；**渲染画 enclosed 房间地板+墙+名**。门：①空 rooms → digest==基线（`Compare-Object`）；②有 rooms → goto_tick 两跑 confide_events + event_digest 一致；③`confide_events` 不归零（软门校准）；④LOD off+on 各验隐私。

**阶段 2 内容数据化**：`data/buildings.json` + `_compile_buildings()`（静态注入、guard、sorted）+ `room_templates.json` + `_hash01` 变体；`owner` 字段接 housing 前哨。门：同 seed 两次编译 objects 逐字节一致；无 `_rng_at`（grep 审）。

**阶段 3 空间裁剪抗规模**：`_objects_by_area` 索引（编译期填 + spawn/despawn 维护，须覆盖节日 fest_ 增删）。门：裁剪前后 digest 逐字节一致；`cand_calls` 内层迭代随所在区家具数而非全图增长。

**阶段 4 渲染升级（纯装饰）**：roof-cutaway 读 `_near_set` + CC0 瓦片替程序化色块 + 楼层选择器。门：切视图/切层不产生 event、digest 不变；goto_tick 截图一致。

**阶段 5 多层终态（A）**：`ag["pos"]`→`location={map,floor,pos}`，楼层独立子网格，楼梯 portal 节点图。门：单层退化等价于 B；跨层寻路无 RNG；goto_tick 跨层重演一致。

## §8 SKIP（红线外/过度工程）
真物理碰撞 · 电梯排队仿真 · 室内破坏 · 真 3D 堆叠（z 坐标是回放杀手）· 租房市场撮合 · 给 agent 加 Vector3i · portal 离屏坐标块（污染 LOD）· spawn_object 造编译期家具（logs event→digest 漂）· 非 CC0 室内素材。

**首切片选定**：阶段 1 —— 给 home 一个 enclosed 卧室 + confide/leak 私密门 + 房间渲染。理由：以最小改动（纯数据 + `_room_at` ~8 行 + 一个 enclosure if + 渲染 pass）给**已在跑的 S3 秘密博弈补上物理正当性**——秘密从"广场当众吐露"变成"关起门说的悄悄话"，这是本项目叙事完整度的一块实缺的地板。

## §9 阶段1 已落地（2026-07-04）
`Sim._room_at(pos)`（`_area_at` 孪生，rooms 独立 dict 无 overlap）+ `_same_enclosed_room` + `ag["room"]` 缓存（`_make_agent`/`_move_agent` 两写点，与 area 同址）+ `_social_candidates` confide/leak 私密门（顶层 `(world.get("rooms",{}) as Dictionary).is_empty()` 数据门）+ `data/map.json` 加 `home_bedroom`(rect[2,2,6,4] 覆盖 bed_1[3,3]/bed_2[6,4]，enclosed)+ WorldView 画 enclosed 房间（木地板+墙框+🚪名，纯渲染）+ soak 室内隐私诊断。
- **门全绿**：①rooms 缺失 → S0 12 seed digest 与改前基线**逐字节相同**（`Compare-Object`）；②`betray` 场景 rooms ON → 吐露心事=1/说漏秘密=3（**>0 未因隐私门饿死**）+ 36/36 不变量；③默认场景 rooms ON（无种子秘密→隐私门不触发）= 基线逐字节（外科式）。
- **坑（docs/13）**：`var priv_ok := world.get(...).is_empty() or ...` 编译失败——`world.get()` 返 Variant，`:=` 推不出 bool；`(... as Dictionary)` + 显式 `: bool` 修。另：多 docker 容器并发跑同一挂载卷 + 编辑期改文件 = 交叉读半成品 → 假 SCRIPT ERROR；纪律=改完再验、单容器串行、验证期不碰文件。
- **诚实局限**：confide 计数偏低（卧室人流稀 + 场景只 1 个种子秘密）——够证"私密门不饿死"，但要秘密博弈真活跃需后续让 agent 主动"约到私密处说心事"（阶段2 家访/私宴 或给 confide 加"寻私密"倾向）。多层/roof-cutaway/buildings.json 编译展开 = 阶段2+。

## §10 秘密博弈激活 + 隐私门广义化（2026-07-04，承接 §9 诚实局限）

**动因**：§9 留了"confide 稀少"的尾巴。一次家访(visit)dead-end（docs/13）证伪了错误瓶颈后，量化定位真瓶颈=**日常里根本没有新鲜秘密可说**（秘密只在 betray 场景种）。

- **`data/secrets.json`（私密秘密种子）**：`{seeds:[{owner,claim}]}`。`start_new` 里**仅默认场景(`scenario==""`)**给每个存在的 owner 一条 self-subject 秘密（`S_own_<id>`，`secret/owner=self/via=seed/confidedBy={}`）。缺文件=零播种=逐字节不变（off 门）；定向场景(betray/faction/freerider)不播种、保持纯净可机检。6 名角色各一条私密（离镇看海/怕黑/暗恋/识字少/欠债/假名）。
- **`_secret_private(ag,o)`（隐私门·广义版，替换 §9 的"仅同封闭房间"）**：`rooms` 缺失→恒 true（旧行为=逐字节不变）；否则 **同一封闭房间(墙保私密，闹市也算)** 或 **独处(说话者 `EARSHOT=2` 曼哈顿格内无第三者)**。理由：稀疏地图上"仅同封闭房间"把 confide 卡死（全镇 1 卧室）；"无旁人偷听"既解卡又更真实，且墙仍有意义。纯读 area/room/pos 缓存 → 确定性。
- **量化纪律（吸取 dead-end 教训）**：soak 加"关系就绪对（有秘密+trust≥25&aff≥10，忽略隐私）vs 实际吐露"探针——播种后测得 6 就绪 vs 1 吐露，一句话锁定"卡在隐私/同室共处不在关系建立"→ 才动隐私门；广义化后 1→2，再测仍稀少 → 定位**更深瓶颈=agent 无"和信任者私下独处"的动机**（visit 想解、解错的那个问题的正确版本）。
- **门全绿**：①`secrets.json` 缺失 → 默认沙盘 confide=0、36/36 不变量、`_secret_private` 被算但无秘密可门=纯读无副作用 → 逐字节等价基线（off 门）；②`secrets.json` 在 → S0 **12 seed 全绿 + 36/36（硬+软）+ 确定性两跑一致 3/3**；秘密专道 #21 / 背叛可溯源无误判 #22/24 全过。
- **诚实局限（下一杠杆）**：默认沙盘 confide 仍稀（多数种子 0–2、betray=0）。秘密稀少本就合理（罕见才有戏剧性）+ betray 场景永远能按需演全弧。要日常里更活跃 = 给"信任好友私密独处时光"一个动机（新机制/设计决定，单列，不再对着想象瓶颈瞎加）。
