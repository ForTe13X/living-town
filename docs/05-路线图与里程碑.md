### M1 · 确定性生活模拟（肌肉，零 LLM）
**目标：一座 6–10 NPC 的小镇照常生活、好看、可加速/暂停，完全不依赖任何模型。**
- 需求向量 + 每 tick 衰减；世界对象「广告」交互；效用 AI 从 `agent_candidates()` 选 option。
- option-action 分层：option 持续、低层动作脚本执行。
- tick(60Hz)/step(~1s) 调度；像素地图 + Agent 精灵 + 寻路 + 头顶需求/动作气泡。
- `sim_soak.gd`：跑 30 天，断言无饿死/卡死、需求在健康带、社交发生过 → 平衡纪律就位。
- **验收**：浏览器/桌面可玩的「会自己过日子的小镇」；headless soak 通过。

#### M1-social · 确定性社交底座【垂直切片，已落地】✅（2026-06-27，见 docs/06）
> 两轮评审（内部 7 维 + GPT-5.5 Pro）一致判定：原骨架最致命缺陷是「引擎无任何 Agent↔Agent 交互」。
> 据此先把社交做成**一等的确定性能力**（先于 M2 LLM）。本切片已实现并经 `tools/sim_social_port.mjs` 跨 5 seed + 60 天验证。
- `agent_candidates()` 除物件外，枚举【同区可感知】其他 agent 的社交动作 **greet / give / gossip**。
- **SocialTransaction** 协议：发起 → `_acceptance_rule` 评估(接受/拒绝) → 提交 → 双方+旁观者各写**视角不同**的记忆。
- **关系账本** `relationships{affinity,familiarity,last_pos,last_neg}`（每次变化溯源 event id）。
- **belief / 知识边界**：claim 只能经 gossip 事务或观察获知（种子谣言 R1 靠 gossip 扩散）；无来源 belief = 0。
- **不可变 `event_log`**（replay/debug/bench 根）；`option` 带 `partner` + 多 tick 对话相；`_rng_at` 接入平局抖动（修死代码）。
- `sim_soak.gd` 升级为**社交不变量门**（7 条断言 + nonzero 退出）：无饿穿 / 社交发生 / 无永久孤立 / 关系分化 / 谣言传播 / 知识边界 / 账本可溯源。
- **已在真 Godot 4.6.2 跑通**（复用 22nd 已构建的 `gamecraft-runner:4.6.2` 镜像、独立一次性容器、`tools/soak-godot.ps1`，不碰 22nd 的 pipeline/容器）：determinism（同 seed 双跑一致）+ 多 seed 不变量全过、退出码 0。真引擎逼出并修掉三个 headless 专有问题：`max()` 返回 Variant 的 `:=` 类型推断、`class_name` 全局类未导入（改 `preload`）、Sim 前向引用晚注册的 AIBackend（**已解耦**：`Sim.backend` 可注入，Sim 不再引用任何 autoload 全局名，引擎可独立 headless 实例化）。

#### M1-commit · 承诺系统【已落地】✅（2026-06-27，GPT-5.5 §7.2 点名的「戏剧来源」）
- 新社交动作 **`invite`**（约对方稍后在【当下同区】再聚）→ 创建 **`meet` 承诺** `{id,a,b,area,created,deadline,status}`；按熟悉度加权（约熟人、非陌生人）。
- 引擎给 **`attend`（赴约）** 行为加权：临近 deadline 主动前往约定地；但**需求危机**（任一 need < `NEED_CRISIS`）会让人**放弃赴约 → 爽约**（确定性的「危机即背叛」之源）。
- 每 tick `_resolve_commitments()`：双方按时到场 → **fulfilled**（trust↑/affinity↑）；到点缺席 → **broken**（按谁缺席**归责**：trust↓/resentment↑/affinity↓ + meet 违约事件 + 双方视角记忆）。关系账本加 `trust`/`resentment`。
- soak 增 3 条承诺不变量（共 **10 条**）：承诺生命周期(创建>0 且兑现>0)、无悬挂承诺(已过点必结算)、违约可溯源(每个 broken 有违约事件 + resentment 出现)。
- **真 Godot 验证**：45 天多 seed → 创建 81–95 / 兑现 79–93 / **爽约 1–3** / 待办若干，10/10 全过、determinism 一致。先经 Node 端口跨 5 seed+60 天验证。
#### M1-conflict · 冲突生命周期【已落地】✅（2026-06-27，GPT-5.5 §9.3）
- 怨气来源：**爽约**(+5)与**被婉拒**(+3) 经 `_bump_resentment` 累积；过阈值(`CONFLICT_TRIGGER`)→触发一段冲突 `{a 委屈方, b 冒犯方, status, severity, escalations, confronted, repaired}`，状态机：**simmering→escalated→confronted→repaired / lingering**。
- 新社交动作 **`confront`（当面对质）**：B 接茬→`confronted`（通往和解）；B 否认/回避（寡言者更易）→`escalated`(severity↑/resentment↑)。**`apologize`（道歉）**：**仅在被对质后**才会发起（知识边界——冒犯方被挑明才知错）；A 视 `severity` 决定原谅→`repaired`(清账、trust/affinity 恢复) 或拒绝（留待下次）。久未对质→`lingering`(冷战)。
- soak 增 3 条冲突不变量（共 **13 条**）：冲突生命周期(触发>0 且 对质/修复>0)、先对质后和解(repaired 必先 confronted)、修复可溯源(每个 repaired 有被接受的 apologize 事件)。
- **真 Godot 验证**：45 天多 seed → 触发 6–26 / 对质≈触发 / 修复多数、含"未被原谅"分支(seed42=6/6/3)与 lingering(60天偶发)；13/13 全过、determinism 一致。先经 Node 端口跨 5 seed+60 天验证。
- 现在小镇能长出完整因果链：约见→爽约→积怨→（成冲突）→当面对质→道歉→和解（或拒绝/冷战），全部确定性、可 soak、零模型。

- **下一步（按 GPT-5.5 §7.2/§9）**：SocialTransaction 字段化(preconditions/effects)、Topic 对象、类型化反思、还礼/秘密类承诺；扩到 15 个社交动作；接 WorldView 渲染 `social_event`（气泡/关系连线/冲突标记）；写 living-town 的 bench task 接进 22nd 评审；然后才进 M2 接 LLM 对话。

### M2 · 接 LLM 对话（声音，玩家↔NPC）🟡 异步接线已落地+mock 验证（真模型待本机联调，见 docs/07 §12）
- 复用 22nd 三档后端：`logic`(默认) / `llm`(LM Studio) / `slm`(godot-llm 内置) / + `mock`(验证)。✅ 已实现异步生命周期(`_wait`/兜底/`thinking`/在飞上限/deadline) + `parse_decision` 健壮 pick glue + json_schema；`scenes/m2_test.tscn` 自测过。
- 待办：玩家自由文本输入框 UI、流式回应、真模型联调（NPC↔NPC 已可由 mock/真模型自动驱动）。
- 玩家走近 NPC → 文本输入 → 模板 prompt(PERSONA/WORLD/NOW) → 流式回应 + 「思考中」气泡。
- 结构化输出：`{speech,emotion,action,target,affinity_delta}`，enum + 语法约束 + 代码校验 + 引擎兜底。
- 异步 `HTTPRequest`，`thinking` 标志，~400ms deadline 降级到罐头台词。
- **验收**：装/不装模型都能玩；装了能和 NPC 自由对话且行为合法；无卡帧。

### M3 · 记忆与反思（活人感）
- `Memory.gd`：append-only 记忆流 + recency/importance/relevance 算术检索（先不用 embedding）。
- 对话/重要事件写入记忆；NPC 回应注入 top 3–5 条记忆 → 记得你/记得彼此。
- 夜间**反思**：把当天记忆 LLM 提炼成洞察写回；summarize-and-forget 控上下文。
- **验收**：NPC 隔天记得昨天的事；记忆面板可看；上下文长度有界（与时长无关）。

### M4 · 涌现社会（灵魂）
- NPC↔NPC 相遇生成简短对话、互写记忆；好感/关系图演化。
- 轻量 `Director`：偶发设定镇上意图（八卦、心情、点名接近玩家）。
- 玩家介入：送礼/撮合/办活动 → 经由对话与记忆**自行扩散**（致敬 Smallville 派对涌现）。
- PIANO 式并发：快反射(脚本)/慢决策(LLM)/反思(LLM) 不同节奏，CC 保「说/做」一致。
- **验收**：出现非脚本的社会小戏剧；每周目不同；成本受控（导演-Agent 分层 + 缓存）。

### M5 · 评测 / 平衡 / 打磨（复用 22nd 流水线）
- 把工程接入 22nd `pipeline`：写一个 bench task（instruction + rubric + scenario demo），用 **LLM-as-judge** 自动评「小镇看起来活不活、需求/社交对不对」。
- soak 仿真扩展到经济/关系；一次调一个旋钮看健康带。
- 美术三级回退：先占位像素图（svg/gen png），后零代码换正式像素美术（Aseprite）。
- 安全过滤层（玩家可见文本上线前）。

### M6 · 发布
- HTML5/wasm（`logic` + 可选远程）+ 桌面（内置 SLM）双轨，复用 `serve.js` 与 exe 导出。
- 可选打磨：语音 STT/TTS（Mantella 路）、建造/经济沙盒、更多 NPC/地图（S/M/L 内容分档）。
- 逃生舱：若 Agent 数走向上千 → 评估把热循环移 C#，极端再看 Bevy。

## 成本/性能护栏（贯穿全程）
- 每-NPC 每游戏小时**硬性 LLM 调用预算**；并发 1–2；优先队列；`max_tokens≈80`。
- 先把脚本 AI 做好做平衡，LLM 永远是渐进增强。
- bench 红线：scenario/回放/soak 永远确定性、零模型。
