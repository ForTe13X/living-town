# 04 · 参考项目

> 给「像素版模拟人生 + LLM」的设计提供经过核实的对标。**抄什么、向谁学**一栏给出可操作结论。
> 学习这些项目用于研究与设计参考，**不复制其代码/美术资产**；自研实现以避免侵权。

## A. LLM-Agent 生活/社会模拟（玩法与 Agent 架构的灵魂）

| 项目 | 是什么 | 抄什么 / 向它学 |
|---|---|---|
| **Stanford Generative Agents（Smallville）** [arXiv:2304.03442](https://arxiv.org/abs/2304.03442) · [code](https://github.com/joonspk-research/generative_agents) | 25 个 LLM Agent 在 Sims 式像素小镇里规划一天、交谈、结识、自办情人节派对 | **可信度三件套，近乎照抄**：append-only **记忆流**；检索按 **recency(衰减)+importance(LLM 评 1–10)+relevance(embedding)**；**反思**（周期性把记忆 LLM 提炼成更高层洞察、写回，可再反思成树）；**世界状态树**(world>area>object 状态串，Agent 只部分观察)。**警示**：全量 25-agent 2 天跑掉数千美元 → 成本是头号工程问题 |
| **a16z AI Town** [github](https://github.com/a16z-infra/ai-town)（~10k★，MIT） | Smallville 的**可发布**实时游戏版（JS/TS + Convex）。**最该先读的工程代码** | **tick/step 批处理**（60Hz 内存跑、~1s 持久化一次，别每帧写盘）；**LLM 调用与 tick 解耦**为异步内部动作、结果回灌为游戏输入（1–2s 延迟不卡顿）；**只经提交的 input 改状态**（单序列化点、可回放）；对话**摘要+embed** 入向量库 + **embedding 缓存**（按文本 hash，省钱）；活动状态保持很小；模型可换（含本地 Llama3） |
| **Altera Project Sid（PIANO）** [arXiv:2411.00114](https://arxiv.org/abs/2411.00114) · [code](https://github.com/altera-al/project-sid) | Minecraft 里 10–1000+ Agent 形成经济（宝石货币）、投票通过宪法、传播宗教、分化职业 | **PIANO**：模块**并发跑在不同时间尺度**、共享一个 Agent State，慢思考不阻塞快反射；**认知控制器(CC)** 经信息瓶颈定高层动作并**广播**，让「说/做」一致。→ 多 Agent 实时**tick 调度**与一致性的模板，**绝不每 tick 跑单条庞大 LLM 链** |
| **Voyager** [arXiv:2305.16291](https://arxiv.org/abs/2305.16291) · [code](https://github.com/MineDojo/Voyager) | NVIDIA 等的 Minecraft 终身学习 Agent：自动课程 + 技能库 + 自校验代码生成 | **技能库即记忆**：把学会的多步例程存成**命名可调用代码**，复用时确定且**免费**（无 LLM 调用）——重大省钱杠杆 + 学得策略缓存；**自校验循环**：任何 LLM 动作先对世界校验、错了重试，别信原始输出 |
| **Lyfe Agents** [arXiv:2310.02172](https://arxiv.org/abs/2310.02172) | 为低成本实时社交而工程化的 generative agents，**报告 10–100× 降本** | **成本生存包**：**option-action 分层**（LLM 偶发选高层 option，低层动作脚本廉价执行——按决策而非帧调模型）；**异步自监控摘要**；**summarize-and-forget 记忆**（聚类/摘要/丢弃低价值，检索成本有界）。配 [Affordable Generative Agents](https://arxiv.org/abs/2402.02053)：用缓存/学得策略替代重复推理 |
| **Skyrim Mantella / CHIM·MinAI** [github](https://github.com/art-from-the-machine/Mantella) | Skyrim/FO4 NPC mod：STT(Whisper)→LLM→TTS，带持久记忆、事件感知、动作触发，**可全本地离线** | 证明 **STT→LLM→TTS→动作** 链能在已发布游戏 + 业余预算上跑、**本地 SLM 可行**。实践：把实时游戏事件作为新「观察」注入(grounding)；LLM 出**结构化动作命令**由引擎校验执行；跨会话持久对话记忆；**把 LLM 约束在受校验的动作词表**防幻觉 |

补充书目：CivAgent（Unciv 里的 LLM 玩家）、SimsChat（人设角色扮演框架）、AgentSociety（万级 Agent 社会）、[awesome-LLM-game-agent-papers](https://github.com/git-disl/awesome-LLM-game-agent-papers)（活体文献）、**Richard Evans**（The Sims 3「个性建模」GDC + Versu/Sketches of Life 的符号社会实践 AI——学怎样**廉价地**建模需求/关系，再让 LLM 叙事；别用 LLM 干状态机免费能干的事）。

## B. 生活/殖民地模拟（系统设计灵感，非 LLM）

| 项目 | 抄什么 |
|---|---|
| **The Sims** | 经典**效用-AI 需求模型**（饥饿/社交/趣味/精力「motive」→ 对象「广告」交互）。当确定性骨架；LLM 是 Sim 的**心声/对话**，绝不是合法性 |
| **RimWorld** | **Defs(数据=是什么) + C#/Harmony(行为=怎么做)** 的黄金范式；需求/心情→动作 + 「讲故事人」涌现。→ Godot 里用 JSON/`.tres` 村民/物品/事件 Def + GDScript 行为，LLM 叠叙事 |
| **Stardew Valley** | 日/季**时钟+日历**循环、NPC 作息与好感心数、**数据驱动内容 + Content Patcher/SMAPI** mod 生态。→ 内容（作物/NPC/事件/对话）作可编辑数据文件，别硬编码 |
| **Project Zomboid** | **Lua 即 mod 语言** + 深度需求/状态（饥饿/心情/伤）。→ 早期暴露脚本/数据缝给 modder |
| **Dwarf Fortress** | 仿真深度与程序化历史；同时是 **ECS/性能轴**的警示（上千 Agent 才需要 Rust/Bevy，本作不需要） |
| **Ooblets / UE 生活模拟** | 反例：重 3D 引擎对俯视像素是杀鸡用牛刀，会撑大体积、毁掉廉价 HTML5 路 → 坚持轻 2D（Godot） |

## 综合：本作应当построить 的东西

- **记忆流 + recency/importance/relevance 检索 + 反思树**（Stanford）—— 可信度核心，近乎照抄。
- **tick/step 批处理 + LLM-as-async-input + embedding 缓存**（AI Town）—— 工程核心，先读其代码。
- **PIANO 并发模块 + CC 瓶颈**（Project Sid）—— 多 NPC 实时一致性与 tick 调度。
- **option-action 分层 + 缓存/学得例程 + summarize-and-forget**（Lyfe/Voyager）—— 成本核心，让它可发布。
- **钉在世界树上的受校验动作词表**（Stanford 树 + Mantella 动作命令）—— LLM 输出永远可执行、不幻觉。
- **分层 人设/记忆/情绪/安全**（Inworld 式）+ 玩家可见文本上线前过一道安全过滤 —— 生产卫生。
