# docs/19 · 大地图设计 Roadmap（分区 / 流式 / 跨区社会传播 / 确定性）

> 目标：把《小镇有灵》从**单张 24×16 固定网格**扩到**可漫游的大地图**（多街区/多村镇），而**五条红线一寸不让**：
> ① 确定性 + 逐字节回放（`goto_tick`+`Invariants.digest`+`event_digest`）② 无模型也可玩（logic 地板默认）
> ③ 消费级硬件（含手机）④ 无版权风险（CC0/自绘/程序化）⑤ 复用优先。
> **本文是设计 roadmap，不是已建成的东西。** 姊妹项目 `Dev/Jianghu`（同为 Godot4 确定性 headless + 信号 UI 的 wuxia 开放世界）已把「世界瓦片」这套冻结成 ADR，本文把它的可借之处**按小镇的红线重新适配**后落成分阶段计划。

---

## 1. 现状：小镇已经就位的"可扩展地基"

扩大地图不是从零起，`Sim.gd` 里这些**已验证**的东西正是大地图的地基（复用优先）：

| 能力 | 现状（file） | 大地图为何用得上 |
|---|---|---|
| 单一世界状态 + 信号驱动渲染 | `Sim.gd` 世界字典 + `WorldView` 纯订阅 | 视图与仿真解耦 → 只渲染"活跃"区、仿真照跑 |
| **LOD 分级** | `lod_near_radius=8` / `lod_far_mult=3` / `lod_aggregate` / `lod_near_cap`（`Sim.gd:85-88,139`）| 远端 agent 降频/只维持统计——大地图的核心省算力手段，已冲到 ~N=1000 |
| **LOD 焦点是 sim 内部点** | `lod_focus := Vector2i(12,8)`「headless 默认镇中心」（`Sim.gd:140`）| **决定性焦点**（非人眼相机）→ 见 §4 关键适配 |
| 确定性 RNG（子流抖动） | `_rng_at(salt,who)`（`Sim.gd:211`，`seed_base+tick*911+salt*7919+who*104729`）| 大地图要"每瓦片一条 RNG 流"防加载顺序扰动 digest（§5）|
| 不可变事件账本 + digest | `event_log` + `Invariants.digest`/`event_digest` | 回放红线的锚；瓦片流入流出不能破它（§5）|
| tick/step 异步 | `TICKS_PER_DAY=240`，tick 推进 + step 检查点 | 大地图加"瓦片级经济 tick"直接挂 step |
| 数据驱动 = 加系统 | `data/<x>.json` 缺文件逐字节不变（经济/天气/职业/节日/技能/住房/选举/生命周期）| 新瓦片/区域内容走同款 data 门，off 时零扰动 |
| 场景 provider + 扩展中枢 | `DataScenarioProvider` / `SimExtensions`（5 挂点）| 瓦片生成/进出可做成 provider，不动核心 |
| 跨平台整数确定性 | 零 transcendental + FNV-1a + PCG32 | 瓦片种子派生用同款整数哈希 → 跨端同图 |

**结论**：小镇的社会仿真（S/L 系列 + 三波世界系统）已经比江湖的当前原型更"接近可规模化"；大地图应当**扩展**这些，而不是重建。

## 2. 江湖姊妹项目可借的（已冻结成 ADR）

`Dev/Jianghu`（Godot4 + GDScript，确定性 headless + 信号 UI；见其 `PROJECT_CHARTER.md`、`docs/phase0/*`、`docs/adr/0002/0003`）把开放世界这套想清楚了：

- **三层地图**（ADR-0003）：**世界瓦片网格**（20×20，战略/经济状态容器，~1 瓦片/游戏日）→ **局部格子地图**（进入瓦片时按 `seed+context` 程序生成或手作）→ **微场景**（室内/洞窟，手作 prefab）。
- **可见性=信息门，非等级门**：`unknown→rough→known→insight`，靠买地图/谣言/情报/侦察/跟商队解锁；玩家能早进高危区，靠**准备与情报**而非属性门。
- **状态优先确定性**（ADR-0002）：唯一 `GameState`，一切改动走 `Reducer`（纯函数、深拷贝、无副作用）；LLM 产出是**意图**不是状态突变。
- **RngService 命名子流**（`world/tile/encounter/battle/loot/agent_variation`）：每系统一条流互不扰动，`snapshot_cursors()` 存游标 → 回放。
- **持久 delta**：不存整张瓦片，只存"相对种子基线的改动"（拿走的苹果/交的朋友/烧掉的建筑）；未改的从种子重生。
- **跨距经济传播**：市场节点（3-5 个区域枢纽）+ 显式贸易路线邻接图 + 谣言扩散带衰减+最大传播距离——**影响不瞬时全图**，一次只读相连瓦片的事件。
- **SaveManager**：信封 `{save_version, state_hash, state}` + 原子写 `.tmp`→轮转 + 三槽回滚 + 哈希校验 fail-loud。

**但江湖是「单人行者」范式**（观察者=玩家，天然以玩家为中心做 LOD/加载）。小镇是**自治小镇**——这一条差异是全部适配的关键，见下。

## 3. ★核心适配：自治小镇 ≠ 单人行者（观察者无关的 LOD）

**江湖能"只精细模拟玩家所在瓦片"，因为它的世界就是绕玩家转的。小镇不能照抄——否则会同时踩两条红线：**

1. **确定性/回放**：若"精细模拟哪块"取决于**人眼相机**在看哪，则小镇的历史（谁跟谁吵架、谁传了谣言）就成了**观察路径的函数**——同存档不同看法回放出不同 `event_log` → digest 不可复现，回放红线破。
2. **headless soak = 窗口**：CI 用 `Sim.backend=null`、`lod_focus`=镇中心跑；若窗口把 `lod_focus` 挪到相机，窗口历史就和 CI 金标不一致。

**解法（小镇专属）：仿真保真度由【决定性规则】分级，不由人眼分级。**
- **LOD 焦点恒为 sim 内部决定性点**（镇中心 / 或一条决定性的"戏剧焦点"轮询规则），**不接人眼相机**。渲染可以只画相机所在区（省 GPU），但**仿真的分级与相机无关**。
- 大地图上：**所有瓦片都被仿真**，只是分级——活跃瓦片全保真、邻接瓦片降频、远瓦片只跑"日程级"轻 tick、极远只维持摘要。**分级依据是"到决定性焦点的瓦片距离"**，不是相机。
- 规模来自"远瓦片的轻 tick 足够便宜（O(N) 线性维持，同现有 `lod_aggregate`）"，而非"跳过没在看的瓦片"。

> 一句话：**江湖的 LOD 是"玩家在哪精细哪"；小镇的 LOD 必须是"决定性规则说精细哪就精细哪"**，这样镇子无论有没有人看、跨不跨端，都演同一台戏。这是把江湖世界瓦片模型搬进小镇时**唯一必须改的地基假设**。

（若日后要做"玩家作为居民漫游"的可玩模式，可另开一条**观察者相关的"游玩视图"**，但 CI/回放金标永远跑观察者无关的 canonical 路径——两套 LOD、一套真相。）

## 4. 确定性红线的接法（大地图最难的一寸）

digest 只哈希 `event_log` 结构（`id:type:actor:target:accepted:subject:tick`）。大地图新增"瓦片加载/卸载/生成"这些动作，必须做到**回放逐字节一致**：

1. **每瓦片一条命名 RNG 流**（借江湖 `RngService`）。当前 `_rng_at` 的平局抖动盐=候选下标（"候选扰动陷阱"：加一个候选就扰动整条盐流）。瓦片尺度上同理：**若所有瓦片共享一条 RNG，则加载顺序/时机会扰动流** → 回放随观察漂移。改为 `tile_rng = stream("tile:"+coords)`，每瓦片生成/事件用自己的流，**加载顺序无关**。
2. **瓦片从 `world_seed + coords` 决定性生成**（FNV-1a，同现有整数哈希）。同瓦片进→出→再进 = 同布局同 NPC 同随机（不重掷）。
3. **持久 delta 回放**：瓦片状态 = `种子生成基线 + deltas`。存档只存 deltas，加载=重生基线+叠 delta。deltas 必须**合并/抵消**（拿起又放回=空），否则加载慢 + 顺序敏感。
4. **瓦片进出不写 event_log（或只写决定性事件）**：远瓦片轻 tick **不产生社交事件**（只维持需求统计，同 `lod_aggregate`）；只有活跃+邻接区产生进 digest 的事件。焦点既然决定性，事件序列就决定性。
5. **off 门逐字节不变**：大地图整套走 `data/world.json` 缺文件回落单瓦片（=今天的 24×16）；`world.json` 不在 → `WORLD_GRID=1×1` → digest 与今天完全一致。同"加 JSON=加系统"范式。
6. **新增硬不变量**：瓦片守恒（NPC 总数跨加载不变）、delta 可溯源、`world_seed+coords→布局` 两跑一致（工具自检，同 scriptwriter 的双跑 digest 对拍）。

## 5. 分阶段 Roadmap（每阶段 off 门逐字节不变、可 soak、真机眼验）

**Phase A · 世界瓦片壳层（WorldState 之上，`Sim` 之下）**
- `Sim` 上加 `world_tiles: Dict[coords→TileState]`（轻状态：NPC 归属/情绪/资源池/发现度）；当前 `GRID 24×16` = "活跃局部地图"。
- `data/world.json`（缺→`1×1`=今天，逐字节不变）。瓦片种子 `= fnv(world_seed, coords)`。
- 尚不做进出，先让"1 瓦片"跑通壳层且 digest 不变 → 证明地基无扰动。

**Phase B · 跨瓦片 NPC 日程 + 观察者无关 LOD 分级（§3）**
- `TileState.npc_schedules`（家瓦片 + 作息 [起床/上工→目标瓦片/回家/睡]）。
- 缺席 NPC 按日程**轨迹插值**维持（不逐帧）；进其瓦片时按"日程时刻"决定性重建。
- LOD 分级依据"到决定性焦点的瓦片距离"（活跃全保真 / 邻接降频 / 远日程级 / 极远摘要）——**扩现有 `lod_far_mult`/`lod_aggregate` 到瓦片粒度**。

**Phase C · 跨区社会传播（复用现有 gossip/Ledger，加"距离与衰减"）**
- 谣言/声誉沿**邻接图**扩散，带**每日衰减 + 最大传播距离**（防瞬时全图，呼应 docs/10 "私有声誉小 N 别瞬时收敛"）。
- 经济：**商品组级**（8-10 类）+ 市场节点（3-5 枢纽）+ 贸易路线；价格=公式(supply/demand, route_risk, ...)，复用 `Ledger`。事件→`WorldDelta`→只读相连瓦片。

**Phase D · 持久 delta + 存读档（借江湖 SaveManager）**
- 信封 `{save_version, state_hash, tiles:{coords:{deltas:[]}}}` + 原子写轮转 + 哈希校验。
- **跨加载 digest 门**：存→读→继续跑，与不存读的连续跑逐字节一致（这是 Phase D 的验收硬指标）。

**Phase E · 流式（手机内存约束）+ 可见性信息门（复用 belief/gossip）**
- 只驻留 活跃+邻接 瓦片，远瓦片压成摘要卸载 → 内存有界（红线③手机）。
- 瓦片"发现度" `unknown→rough→known→insight` 用现有 belief/gossip 解锁（听说→去过→熟络），**信息门非等级门**。

## 6. 陷阱（江湖的 + 小镇专属）

- **别启动即全生成**（20×20×局部 = 数百万格 + 启动寻路 = 💥）→ 首次进入惰性生成、一瓦片一次。
- **别全精细模拟远端 NPC** → LOD 分级（活跃/邻接/日程/摘要），但**分级须决定性**（小镇专属，§3）。
- **别用 `randf()` 生成瓦片** → 整数 RNG + 每瓦片命名流（跨端同图 + 加载顺序无关）。
- **别让 delta 爆炸** → 合并/抵消，只存净改动。
- **别把可见性做成内容门** → 瓦片状态永远 = 种子+delta 决定，可见性只改 UI 信息层，NPC/物不因"没看见"而生灭（否则破确定性）。
- **别把经济做成每 NPC×每物** → 商品组 + 市场节点 + 路线邻接。
- **别把 LLM 放热路径**（每 NPC 选下一个瓦片都问模型=AI Town 陷阱）→ 高频决策走 logic/util 地板，低频（策略/对话/长期规划）离线/异步 LLM（呼应 70B 夜间编剧：不确定性离线冻成数据）。
- **小镇专属·别把 `lod_focus` 接相机**（§3）→ 一破全破（回放 + headless=窗口）。

建议 **Phase A 先行**（世界壳层 + `world.json` off 门 + 瓦片种子），因为它**只加壳、digest 必须逐字节不变**——这一步绿了，就证明大地图能长在现有确定性地基上而不破红线；之后 B/C/D/E 各自可 soak、可眼验、逐个推进。与江湖保持"共享地基、各自演化"（其世界瓦片/经济/SaveManager 是正典，小镇回馈观察者无关 LOD + 社会仿真）。

## 9. 室内即"另一种更大"(Micro Scene) + 复用优先（2026-07-05 用户点拨）

**「变大」不只横向（更多室外街区），还有纵向：进建筑 → 一张细节室内小图 + 专属互动**（图书馆读书 / 柜台冲调 / 诊所看诊 / 工坊修理）。这正是三层里的 **Micro Scene** 层（§2），而且往往**比铺一张大室外图更便宜**——用"可进入的细节室内"换深度。小镇**已有确定性同格室内地基**（docs/16 `buildings.json` 把房间编译进同一张 world 网格 + `enclosed` 隐私门），所以最高 ROI **不是**另造一套 scene-switch 系统，而是**把现有同格室内"呈现"做厚**：焦点在室内时相机拉近/揭顶、渲更细的室内、加室内专属动作。

**红线同款纪律（承 §3）**：室内的"生活"仿真必须**决定性、观察者无关**——别做成"进去才活、出来就冻"的房间（否则破回放 + headless=窗口）。两条干净路子：① 室内是确定世界的一部分（同格，现状）、只是**呈现**更好；② 室内做成独立子场景，但它是世界某区域的**渲染视图**、仿真仍在 canonical 世界里跑。避免"冻结的房间"。

**复用优先**（用户以 Google Fable 5 移植《命令与征服·将军》到 iOS、实为复用开源 **GeneralsX**~2000 commit 为例点拨）：动手造前先把问题**拆成子问题**、去开源社区找可复用成果/组件。落到本项目的分工：
- **该复用的商品级问题**：室内/房间切换（Godot 成熟范式：揭顶同图 / scene-switch / 房间信号梯）、瓦片流式/分块、存读档、寻路、**端上 LLM 绑定**。这些先搜 Asset Library / GitHub，别重造。
- **端上 LLM 具体复用点**：现用 **NobodyWho**；备选 **Adriankhl/godot-llm**（另一 llama.cpp GDExtension 绑定）——若 NobodyWho 内置 llama.cpp 太旧、载不动 Nemotron-3 混合 Mamba，换 godot-llm 的新 llama.cpp 是最省的路（对应 Nemotron 讨论）。
- **该自研的差异化内核**：确定性社会仿真引擎 + 五条红线（逐字节回放 / 无模型可玩 / LLM 冻成数据的确定接法）——**没有开源件能给这套**，这是护城河，不外包。

**原则：红线内核自研，红线外一切商品级问题先搜开源。** 见 [[feedback-reuse-first-open-source]]。
