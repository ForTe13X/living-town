# 06 · 独立评审与风险册（2026-06-27）

> 来源：一次 7 维度 / 41 agent 的对抗式架构评审（每条发现都经独立 skeptic 验证，refuted/uncertain 已剔除，留下 25 条 confirmed）。
> 维度：成本/延迟、行为一致性、记忆漂移、涌现可控/可复现、22nd 复用有效性、代码-文档自洽、参考文献准确性。
> 结论先行：**确定性地板真实、干净、可复现；但引擎没有任何「Agent↔Agent 交互」——而这正是产品的全部卖点。**

---

## 0. 总评

「无模型也能玩」对 `logic` 后端成立，candidate/apply 契约是正确的底座。但有**一处结构性缺陷悄悄使整个产品命题失效**：引擎里两个角色的状态无处相关。其余（prompt-cache 顺序、say/do 契约三处打架、反思齐射、deadline-vs-stream 矛盾、死掉的 RNG 纪律）都是**可修的管线**。多数发现是诚实 M0 骨架上的 M2–M4 前瞻风险；唯独社交缺口落在**被宣称已完成**的部分里（docs/02 §2 说该层驱动约 95% 行为），且使「全里程碑确定性 bench 红线」**永远闸不住卖点**。docs/04 参考异常准确，仅两处出处小瑕 + 漏一条正典（Concordia）。

---

## 1. 🔴 最可能翻车处 — 无 Agent↔Agent 交互（critical）

`agent_candidates()`（`game/scripts/Sim.gd:155-174`）只遍历 `world["objects"]` 的 `advertises`，**从不遍历其他 agent**。连锁后果：

- `talk/give/invite/greet` 候选**没有任何生产者**（grep 确认全库只在 schema/注释里出现这些词）。
- `option` 结构（Sim.gd:183-187）只有 `target/need/amount/dur_total`，**无 partner / 轮流发言 / 共享对话相**。
- `affinity{}`（Sim.gd:83）**只初始化，从不写、从不读**；`affinity_delta` 在 schema/prompt 里有，但无代码消费。
- 「社交」需求由**独自**走到 bench_1/counter_1（map.json）满足；`_advance_agent` 的 use 分支（Sim.gd:138-146）只给行动者自己加需求，无第二方。

→ 两个 NPC 可并排坐着各跑「独自社交」option，一句话不交换。在表达不了「关系」的引擎上叠 LLM「声音」= 需求磨子上的装饰性台词。**它不是被标注的 M4 stub，而是藏在被宣称「已完成」的地板里**——这是它区别于其它发现的根本原因。

**修法**：把社交做成一等的**确定性**能力（见 §8 下一步），先于 LLM 层。

## 3. 盲区（评审者自陈可能漏掉的）

- **寻路/空间成本**：曼哈顿距离、无墙/碰撞/A*（Sim.gd:167），agent 出生即叠在物件上；社交目标是会动的 agent → 需要「走向移动伙伴 + 会合」逻辑，未预算。
- **对话生命周期**：加了 partner 之后，对话是多 tick、双向、可中断的状态机（谁说/谁听/何时结束/一方需求危机中途抽离），并发与放弃语义未定。
- **人格漂移**：6 个固定人设跨上千次 LLM 调用是否走样；罐头降级（`_canned_say` 通用、非人设化）在高负载下会否当众出戏。
- **存档/读档整层不存在**：grep 确认无 `step()`/serialize；生活模拟需要持久存档，整层是散文。
- **本地 LLM 中途不可用**（7B 热切被驱逐 / OOM / llama-server 崩）时的玩家对话降级路径未端到端测试。
- **像素美术工作量未排期**：6 NPC × idle/walk/talk/emote + 表情立绘，是与工程正交的进度风险；继承的 rubric 在无真美术时几乎所有项封顶 0.5。

## 4. 参考文献核查（docs/04）

**异常准确**：6 个 arXiv ID 全对（Stanford 2304.03442、Project Sid 2411.00114、Voyager 2305.16291、Lyfe 2310.02172、Affordable Generative Agents 2402.02053）；许可证全对（godot-llm MIT、NobodyWho EUPL-1.2、AI Town MIT）；PIANO/Lyfe/Voyager/Mantella 描述忠实。三处小问题：

1. **漏了 DeepMind Concordia（arXiv:2312.03664，「Game Master」模式）**——它是本作「引擎当动作校验器」契约的学术正典，应补为对标；但**别替换** Stanford+Mantella/AI-Town 谱系，因为本作刻意用更严的「闭候选 + 按下标挑」变体，而非 Concordia 的自由语言事后裁定式 GM。
2. **Smallville 成本「数千美元 / 2 天」** 是未注明出处的社区估算，且当年用 gpt-3.5-turbo 而非 GPT-4 → 软化为「据估计…数千美元量级」。
3. doc 自称「经核实对标」却在 ② 上没做到 → 过一遍出处。

## 5. 该保留的优点（别在修复中弄丢）

- 确定性地板真实、可独立运行、可复现；`logic` 后端兜底纪律正确。
- candidate→pick→apply 契约是正确底座（只是需扩到社交 + 补 `pick` glue）。
- 记忆流的算术检索（recency/importance/relevance，无 embedding）方向对，省钱；`Memory.gd` 已实现，可不引入 embed sidecar。
- 文档与代码诚实标注了 M2 待实现处；参考调研扎实。

## 6. 立即下一步（最高杠杆，先于 M2）

在**写任何 LLM 集成之前**，把社交做成一等的确定性能力并设闸，一个聚焦改动：

1. `Sim.agent_candidates()` 对每个邻近 agent 也枚举 talk/give/invite intent，partner id 当 target。
2. 泛化 `option` 结构加 `partner` + 共享多 tick 对话相；`agent_apply` 像校验物件那样校验 partner（对活 agent 列表）。
3. 完成时写 `affinity_delta`，给**双方**记忆各记一条带人物 id 的 tag。
4. `sim_soak.gd` 断言可机检社交不变量（N 天内每人 ≥1 次交互、affinity 方差 >0、无永久孤立），违反即 nonzero 退出（复用 rubric 的 build_check 钩子思路）。

→ 这把确定性 bench 从「需求磨子闸」变成「涌现闸」，零模型成本证明命题可表达，并给 M2 一个真正的社交候选集。**做完这步再碰 §2 的 #2/#3/#4 管线修复**（有了可 ground 的社交对象后它们才便宜）。

随后顺手清掉的低成本项：补 `pick→candidates[pick]` glue（移植 22nd `_llm_pick`）+ `agent_apply` 校验 need/amount/dur_total；重排 build_prompt 为 `[共享 WORLD/RULES][per-NPC 人设][易变 NOW/记忆/候选]`；importance 改为写入期由引擎信号派生 + dedup；玩家对话与自主决策分两套延迟规则；`_rng_at` 接入平局抖动或下调 docs/02 §6 措辞并删死代码。

## 7. GPT-5.5 Pro 交叉评审（独立，思考 8m53s + 联网核查，2026-06-27）

把 `_gpt55-review-brief.md` + 本风险册的 P0 发现一并投给 GPT-5.5「Pro 扩展」。结论与本内部评审**强烈收敛**（交叉验证通过），并补了若干净新增的硬架构要求。原话：

> 没有 Agent↔Agent 的一等交互，项目不是"LLM 社会模拟"，而是"若干单人 Sims 机器人 + LLM 台词皮肤"。LLM 不能弥补这个洞。LLM 排在第 8，不是第 1。它不是"一个 bug"，而是 MVP 之前必须重打的地基。

### 7.1 双方一致（交叉验证）
- Agent↔Agent 缺口 = **P0 阻塞级**，"社交因果图不存在"：affinity 是死数据、social need 是假需求。
- 评测须做 **causal bench**：状态级不变量优先于台词；多模态/LLM judge 会被漂亮对话骗。
- 实时性必须**分级**（硬实时移动/需求 0 LLM、准实时高层 option 可超时回退、交互实时玩家对话 streaming、后台反思/总结/导演/离屏社交 队列慢跑）；400ms 只配在 enum 里选，不配自由对话。
- 记忆：**event log = 不可变真相，memory/summary = 派生视图**；区分 truth/observation/belief/rumor/interpretation。
- 成本：现实但苛刻——"一场社交一个仲裁调用，不是每 NPC 各调一次"；离屏 NPC 不演戏、只提交摘要事件。

### 7.2 净新增（本内部评审未覆盖，价值最大）
1. **`SocialTransaction` 原子协议**（光加候选不够）：`{actor, target, optional_subject, action_type, topic, preconditions, acceptance_rule, effects_on_actor/target/relationship, memory_writes, public_observers, replay_trace}`。actor 发起 → target 评估(接受/拒绝/反击/忽略) → 引擎提交 → 双方+旁观者各写不同视角记忆；**台词只渲染事务、不得声明未提交状态**（否则"我把鱼送你了"但 inventory 没变）。
2. **知识边界（知道/不知道是硬边界）**：知道 = 亲见 ∨ 被告知 ∨ 公告 ∨ 推理；每条 belief 必须带 source，无 source 不进长期记忆。防"全知蜂群"。
3. **承诺系统**：`{promisor, promisee, content, deadline, status: active/fulfilled/broken/forgiven, stakes}`。背叛/失望/信任/和解多来自承诺而非好感度。
4. **冲突生命周期**：triggered→active→escalated→confronted→apologized→repaired/lingering（否则 drama 是随机噪声）。
5. **话题对象 Topic**：`{id, type: event|person|place|item|plan|rumor|need|secret, known_by, salience_by_agent, taboo_by_agent, related_events}`，让 gossip/邀约/道歉稳定引用同一件事。
6. **关系账本**（非自然语言）：`{affinity, trust, respect, fear, attraction, debt, resentment, familiarity, last_pos/neg_event, unresolved_conflicts[], promises[]}`，每次变化可追溯 event id。
7. **导演只改"机会地形"不写剧情**："摆棋局不改棋谱"——让有矛盾者相遇/提话题 salience/投谣言种子/排日程冲突；结果仍由个体状态+关系+需求+记忆决定。维护张力预算（town_tension/unresolved_conflicts/social_isolation/rumor_pressure/...）。
8. **类型化反思**（非自由散文人格突变）：`{new_belief, confidence, evidence:[event_id], scope, expires_after_days, contradicts}`。记忆晋升只保留：玩家参与/关系跨阈/承诺违约/礼物债务/秘密谣言/冲突和解/稀有事件。
9. **事件溯源回放**：replay 记录 world_seed/scenario_seed/model_id/quant/prompt_hash/schema_hash/retrieved_memory_ids/candidates/selected/raw_llm/validated/fallback_reason/state_delta；开发期 regression 用 **LLM IO ledger 固定回放**，不重问模型；换模型才开 re-eval。配 **scenario compiler**（三角嫉妒/借物不还/秘密泄露/邀请被拒/误会道歉…，每个定义 initial_state + invariants + failure_conditions）。
10. **法务雷（新）**：**Qwen2.5-3B 不是 Apache-2.0，是 qwen-research license**（72B 同）；商用打包前必须法律核查，别把"Qwen2.5 多为 Apache"泛化到 3B。分工：0.5B=barks/enum、3B=玩家对话/近景、7B=命名角色/离线创作、logic=默认+校验+回放。
11. **工程**：别让每个 NPC 各开 godot-llm 节点（每节点一线程、连续 run_* 会 UI hang）→ **NPCs → 中央 AI 请求队列 → 推理 worker → validator → event input**。Web 端不承诺本地 SLM（Godot Web 受 WASM/WebGL2/SharedArrayBuffer/线程限制，且 C# 不能导出 Web）→ Web = logic-first + 可选远程。

### 7.3 GPT-5.5 修订版里程碑（与本册 §6 一致并细化）
- **M1 · 无 LLM 的确定性社交小镇**：`agent_candidates()` 遍历可感知 NPC + 15 个一等社交动作（greet/smalltalk/ask_need/invite/give/ask_for/help/gossip/compliment/insult/apologize/comfort/confront/refuse/leave）+ 双边接受/拒绝 + 关系账本 + belief/谣言 + event log + 模板台词 + replay bench。**验收**：社交需求不能独自满足；gift 必改 inventory+双方记忆+关系；invite 成功生成日程/intent；gossip 只传已知 claim；NPC 不能知道未观察/未听说的秘密；关系变化可追 event id；同 seed 回放一致。
- **M2 · 玩家可见的受限 LLM 对话**：玩家输入→intent parse→legal candidate；NPC 出结构化 JSON；一场社交≤1 次 arbiter 调用；优先队列 + 每-NPC 冷却 + cache + 超时回退；p50/p95/p99 延迟 bench。**验收**：LLM 关掉玩法不坏；超时不卡帧；幻觉不写状态；问 NPC 不知道的事它不会凭空知道。
- **M3 · 长期记忆/类型化反思/导演**：夜间 summary + typed reflection + rumor/secret + director opportunity scheduling + 社会图指标 + 长跑 soak + scenario compiler + 多日 replay bench。**验收**：7 天后关系图仍可解释；无大量无来源信念；导演提高冲突机会但不直接改结果。
- **最易翻车处**：M1↔M2 交界——做出"看起来能聊"的 demo 后误以为社交系统成立，**其实只是 UI 成立、模拟还没成立**。
